Class Overview

This object represents an asynchronous invocation to a controller. An instance of the CallImpl is typically returned by a method call from any generated controller implementation.

Calling dispose() is optional upon completion. Once get() is invoked, the call result is automatically disposed. However, without get() invocation or if exception occured from get(), the caller should invoke dispose(boolean) to free the call result explicitly.

public boolean cancel (boolean mayInterruptIfRunning)

Cancel a queued or executing call. If the call has been disposed, completed, cancelled, or unable to cancel, it will return false. Upon successful completion, this call object will be disposed too.

Parameters
mayInterruptIfRunning Set to true to permit interruption if the call is running; set to false otherwise.
Returns
  • true if cancellation is submitted; false if unable to cancel.

public boolean dispose ()

Dispose this CallImpl without clearing its result from cache. The call must be in SUCCESS, FAILED or TIMEDOUT state. All resources used by this CallImpl will be released. To dispose a queued or executing call, it must be cancelled first. This method is same as dispose(false).

Returns
  • true for success

public boolean dispose (boolean clearResult)

Dispose the Call and optionally clearing its result from cache. The call must be in SUCCESS, FAILED or TIMEDOUT state. All resources used by this CallImpl will be released. The clearResult allows the caller to remove its result from the cache immediately; it is useful to any infrequent calls with large result data. But caution should be taken since it may delete the cached result for other concurrent similar calls.

Parameters
clearResult true to remove its result if exists.
Returns
  • true

public T get ()

Waits if necessary for the request to complete, and then retrieves its result. Be careful when calling this method in the main thread; it will block the main thread. Once the result is returned successfully, the result is automatically disposed and subsequent call to this method will fail with MobileRuntimeException.

Returns
  • The type-safe result.
Throws
HttpCallException If error code received for HTTP request and response.
MarshallingException If the result failed to convert to the type-safe result.
CancellationException if the computation was cancelled
ExecutionException if the computation threw an exception
InterruptedException if the current thread was interrupted while waiting

public Throwable getCause ()

Get the cause of the failure without using get() or get(long, TimeUnit). If the CallImpl has been disposed, it will return null.

Returns
  • null if no error or the CallImpl has been disposed, or the cause.

public Call.State getState ()

Get the state of this invocation. Reliable call states are State#QUEUED, State#EXECUTING, State#SUCCESS, State#FAILED, State#TIMEDOUT or State#CANCELLED. Unreliable call states are State#EXECUTING, State#SUCCESS, or State#FAILED. The State#INIT is the initial transient state.

Returns
  • One of the State values, or null if the call has been disposed.

public boolean isCancelled ()

Returns true if this task was cancelled before it completed normally. If the call has been disposed, it will return true too.

Returns
  • true if this task was cancelled before it completed

public boolean isDone ()

Returns true if this task completed. Completion may be due to normal termination, an exception, or cancellation -- in all of these cases, this method will return true. If the call has been disposed, it will return true too.

Returns
  • true if this task completed or the call has been disposed; false otherwise.
